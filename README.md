# GP-2
# Анализ данных букмекерских контор

## Описание проекта

Проект посвящен сбору и анализу данных о спортивных событиях и коэффициентах букмекерских контор. Цель проекта - провести анализ данных, собранных с различных букмекерских контор, и сделать выводы о том, на что лучше ставить и каким сервисом пользоваться, что может в будущем послужить основой для нового продукта, выдвигающего пользователю наиболее выгодные предложения.

## Структура проекта

### Этапы исследования:
1. **Сбор данных** - парсинг веб-сайтов и использование API
2. **Предобработка данных** - очистка от пропусков и дубликатов
3. **Аналитика данных** - поиск релевантных сравнений и показателей
4. **Стратегические рекомендации** - бизнес-выводы и перспективы

## Источники данных

### Парсинг букмекерских контор
- 10,494 строки с данными о счете прошедших матчей
- 448 строк с информацией о коэффициентах на будущие матчи
- Использование библиотеки Selenium для динамических сайтов

### API данные
- 21,973 строки с информацией о матчах
- Данные за последние 4 дня по различным видам спорта
- Использование спортивного API

## Признаки полученной таблицы

**Sport** - вид спорта (string)

**Tournament** - название чемпионата или лиги (string)

**Player_1** - первая команда/игрок в матче (string)

**Player_2** - вторая команда/игрок в матче (string)

**Event_time** - время проведения матча (datetime)

**Home_score** - результат матча для первой команды (numeric); только для прошедших матчей

**Away_score** - результат матча для второй команды (numeric); только для прошедших матчей

**Coef_p1_wins** - коэффициент победы первой команды (float); только в завершенных матчах

**Coef_draw** - коэффициент ничьи (float); только в завершенных матчах

**Coef_p2_wins** - коэффициент победы второй команды; только в завершенных матчах

## Выводы

1. Не всегда игра "дома" дает преимущество -> необязательно команда-гость проиграет

2. BetBoom предлагает более выгодные коэффициенты в большинстве случаев -> стоит использовать этот сервис

3. Коэффициенты имеют взаимосвязь с счетом прошлых игр -> показывает, что коэффициенты релевантны
